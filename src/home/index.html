Band Apparel

August 2018 to June 2019
This is possibly my largest and most impressive project to date. My high school didn't offer any more computer courses, so I took an independent study with my favorite teacher (of course the computer teacher). I worked year long on an apparel ordering website built from scratch and hosted on firebase.
I want to have a full writeup on this sometime and even post it on GitHub, but I haven‚Äôt spent the time ensuring no private info/keys are in the private repo.
This was cool, and I miss having the free time to do this stuff, I‚Äôm trying to get back into the swing of personal projects because it‚Äôs so fun and helps ground me. Anyway...
This is a full-fledged website offering everything from account creation to submitting your order (not handling payments) .
I learned how to use webpack, npm, and node for this. Everything relating to infrastructure was firebase. I wrote a custom sign-in process using firebase authentication. I used firebase cloud functions for creating user profiles/buckets in firestore, handling sending out emails to users, securely deleting user data, and getting random images for the homepage.
Something super cool was I wrote all this custom javascript for dynamically loading things. I had a hidden dialog box on every page. In firestore I could change anything and make it popup whenever I needed (like rolling out a new feature, or notifying users about something.)
Similarly, on the store page every apparel item was a document in firestore in a collection of items. I could add or change any apparel item because their info was loaded and cached in firestore. So no having to compile a new build in webpack and upload it. If the band director needed something changed I could do it right from my phone! Man I loved working on this.
